О проекте

FANTOM REDACTION — текстовый игровой движок с открытым исходным кодом, написанный на Python. Главная цель проекта — показать, что создавать свои мини-игры можно просто и быстро. Программа состоит из понятных скриптов, разобраться в которых сможет даже начинающий разработчик.

Актуальная версия: 4.0

Что нового в 4.0:
В этом обновлении движок получил два крупных улучшения: виртуального помощника GitFan и систему тем оформления.

GitFan — это встроенный ассистент который понимает команды на русском языке. Он показывает список игр, находит проекты по жанру, рассказывает сюжет, выводит статистику. Больше не нужно помнить цифры меню — просто спросите.

Темы оформления меняют внешний вид движка. Доступны четыре темы: тёмная, светлая, киберпанк и ретро. Выбранная тема сохраняется и применяется при следующем запуске.

Руководство пользователя

Все действия в программе выполняются по выбору пользователя. Программа всегда подсказывает, какую цифру или букву нужно ввести для нужного действия.

Интерфейс

При запуске программы появляется приветственный экран со статистикой созданных игр и списком последних трёх проектов. Интерфейс адаптируется под ширину терминала — работает как на компьютере, так и на смартфоне через Termux или Pydroid. Цвета интерфейса зависят от выбранной темы оформления.

Главное меню

После загрузки открывается главное меню со всеми возможностями движка.

1 — Создать игру:
Запускает мастер создания новой игры. Вы последовательно вводите:

Название игры

Платформу (Мобайл, ПК или Гибрид)

Возрастное ограничение

Уникальный ID (не менее 7 символов)

Жанр и параметры сюжета

На любом этапе можно нажать 0 для отмены и возврата в главное меню.

Доступно 9 жанров:

Герой и злодей — классическое противостояние

Два друга — история о дружбе и выборе

Экшен — сражения, оружие и боссы

Выживание — управление ресурсами и случайные события

RPG — герой, класс, квесты и спутники

Хоррор — страх, батарейки и монстры

Детектив — расследование с уликами и неожиданными поворотами

Earth — война с инопланетным разумом

Средневековье — королевство, урожай и дипломатия

После заполнения всех параметров игра готова. Программа предложит сохранить её и сразу протестировать. Все числовые поля проверяются — ввести текст вместо числа не получится.

2 — Настройки

Раздел с настройками движка:

О нас — версия движка, разработчик, ссылки

Тема оформления — выбор из четырёх тем: тёмная, светлая, киберпанк, ретро

3 — Загрузить игру

Открывает список всех сохранённых игр. Выберите номер — и программа покажет подробную информацию об игре и предложит запустить тестовый режим.

4 — Управление сохранениями

Позволяет просмотреть список сохранённых игр и удалить ненужные. Удаление требует подтверждения — случайно ничего не пропадёт.

5 — Редактировать игру

Открывает сохранённую игру для редактирования. Можно изменить:

Название

Платформу

Возрастное ограничение

ID игры

Все параметры сюжета

Числовые поля при редактировании проверяются. Изменения сохраняются в тот же файл.

6 — Заметки

Встроенный блокнот для разработчика. Возможности:

Создание заметок с заголовком и содержимым

Просмотр списка всех заметок с датами создания

Чтение любой заметки полностью

Редактирование заголовка и текста

Удаление ненужных записей

Заметки хранятся в файле notes.json и остаются навсегда, пока вы сами их не удалите.

7 — GitFan

Виртуальный помощник движка. Встречает пользователя приветствием и готов ответить на вопросы. Доступные команды:

привет — приветствие

помощь — список всех команд

список игр — показать сохранённые игры

жанр [название] — фильтр игр по жанру

сюжет [название] — показать параметры конкретной игры

статистика — количество игр по жанрам и платформам

заметки — список заметок

кто ты — информация о GitFan

спасибо — вежливый ответ

меню — выход в главное меню

GitFan не требует точных формулировок. Фразы «покажи игры», «что у меня есть», «мои сохранения» работают одинаково.

0 — Выход

Завершает работу программы.

Обзор архитектуры

FANTOM REDACTION построен по модульному принципу. Основной событийный цикл обрабатывает пользовательский ввод и направляет команды в соответствующие модули. Игровые данные сериализуются в JSON-файл saved_games.json, заметки хранятся отдельно в notes.json, настройки — в settings.json.

Код разделён на независимые модули с чёткой ответственностью. Добавление нового функционала не требует переписывания существующего.

Структура проекта

main.py — главный модуль. Содержит точку входа, основной игровой цикл, диспетчеризацию команд и функцию after_create() для сохранения и тестирования созданных игр.

ui.py — модуль интерфейса и тем. Предоставляет адаптивную систему отрисовки: slim_box() и slim_menu() с поддержкой цветовых тем. Темы хранятся в словаре THEMES, применяются через ANSI-коды. Ширина терминала определяется динамически от 30 до 80 символов.

save_system.py — модуль управления данными игр. Инкапсулирует всю работу с JSON-хранилищем: load_saved_games(), save_game(), delete_saved_game(), edit_game(), show_saved_games(). Содержит функции валидации: input_int() и input_bool().

notes.py — модуль заметок. Полноценный CRUD с автоинкрементными ID, перенумерацией при удалении, датами создания и изменения через datetime.

gitfan.py — модуль виртуального помощника. Класс GitFan с методами process(), show_help(), show_games(), filter_by_genre(), show_plot(), show_stats(), show_notes(). Парсинг команд на ключевых словах без сложного NLP.

genres/ — пакет игровых жанров. Девять классов в отдельных файлах, каждый с методами create(), test(), load_and_test(). Поддерживают отмену создания через возврат None.

Формат хранения данных

Игры (saved_games.json) — плоский JSON-массив. Обязательные поля: name, platform, genre_type, age, id.

Заметки (notes.json) — плоский JSON-массив. Поля: id, title, content, created_at, updated_at.

Настройки (settings.json) — словарь. Поле theme: строка с названием темы.

Система тем

Темы реализованы через словарь THEMES в ui.py. Каждая тема содержит:

name — название для отображения

border_color, title_color, text_color, accent_color — ANSI-коды цветов

reset — сброс цвета

Функция apply_theme() сохраняет выбор в settings.json. Функция get_theme() загружает настройку при запуске. Все функции отрисовки используют get_theme() для получения текущих цветов.

Система GitFan

GitFan работает на ключевых словах. Метод process() получает строку, проверяет наличие ключевых слов через any(word in text), вызывает соответствующий метод для формирования ответа. Нет нейросетей, нет сложного парсинга.

Ключевые слова сгруппированы по действиям:

Приветствия: привет, здравствуй, хай, йо

Навигация: меню, назад, выход

Справка: помощь, команды

Просмотр: список, игры, сохранения

Фильтрация: жанр, найди

Сюжет: сюжет, описание, расскажи

Статистика: статистика, всего

Заметки: заметки

Обработка ошибок

Файловые операции обёрнуты в try-except. Некорректный JSON возвращает пустой массив. Пользовательский ввод проверяется. Числовой и булев ввод валидируются. GitFan при нераспознанной команде выводит случайную подсказку и предлагает написать «помощь».

Сборка и зависимости

Только стандартная библиотека Python: json, os, time, random, shutil, datetime. Внешние зависимости отсутствуют. Приложение запускается на Python 3.x без установки пакетов, включая мобильные окружения Termux и Pydroid.
