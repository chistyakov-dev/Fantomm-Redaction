FANTOM REDACTION v3.1.4
---------------
User Documentation
---------------
О проекте

FANTOM REDACTION — текстовый игровой движок с открытым исходным кодом, написанный на Python. Главная цель проекта — показать, что создавать свои мини-игры можно просто и быстро. Программа состоит из понятных скриптов, разобраться в которых сможет даже начинающий разработчик.

Актуальная версия: 3.1.4

Руководство пользователя:
Все действия в программе выполняются по выбору пользователя. Программа всегда подсказывает, какую цифру или букву нужно ввести для нужного действия.

Интерфейс:
При запуске программы появляется приветственный экран со статистикой созданных игр и списком последних трёх проектов. Интерфейс адаптируется под ширину терминала — работает как на компьютере, так и на смартфоне через Termux или Pydroid.

Главное меню:
После загрузки открывается главное меню со всеми возможностями движка.

1 — Создать игру
Запускает мастер создания новой игры. Вы последовательно вводите:

Название игры

Платформу (Мобайл, ПК или Гибрид)

Возрастное ограничение

Уникальный ID (не менее 7 символов)

Жанр и параметры сюжета

На любом этапе можно нажать 0 для отмены и возврата в главное меню.

Доступно 9 жанров:

Герой и злодей — классическое противостояние

Два друга — история о дружбе и выборе

Экшен — сражения, оружие и боссы

Выживание — управление ресурсами и случайные события

RPG — герой, класс, квесты и спутники

Хоррор — страх, батарейки и монстры

Детектив — расследование с уликами и неожиданными поворотами

Earth — война с инопланетным разумом

Средневековье — королевство, урожай и дипломатия

После заполнения всех параметров игра готова. Программа предложит сохранить её и сразу протестировать. Все числовые поля проверяются — ввести текст вместо числа не получится, программа предупредит и попросит повторить ввод.

2 — Настройки:
Раздел с информацией о проекте:

О нас — версия движка, разработчик

3 — Загрузить игру:
Открывает список всех сохранённых игр. Выберите номер — и программа покажет подробную информацию об игре и предложит запустить тестовый режим.

4 — Управление сохранениями:
Позволяет просмотреть список сохранённых игр и удалить ненужные. Удаление требует подтверждения — случайно ничего не пропадёт.

5 — Редактировать игру:
Открывает сохранённую игру для редактирования. Можно изменить:

Название

Платформу

Возрастное ограничение

ID игры

Все параметры сюжета

Числовые поля при редактировании проверяются — ввести некорректное значение не выйдет. Изменения сохраняются в тот же файл.

6 — Заметки:
Встроенный блокнот для разработчика. Возможности:

Создание заметок с заголовком и содержимым

Просмотр списка всех заметок с датами создания

Чтение любой заметки полностью

Редактирование заголовка и текста с обновлением даты изменения

Удаление ненужных записей

Заметки хранятся в файле notes.json и остаются навсегда, пока вы сами их не удалите. Удобно записывать идеи для игр, баги и планы по развитию проекта.

0 — Выход:
Завершает работу программы.

Developer Documentation
---------------------------
Обзор архитектуры: 
FANTOM REDACTION построен по модульному принципу. Основной событийный цикл обрабатывает пользовательский ввод и направляет команды в соответствующие модули. Игровые данные сериализуются в JSON-файл saved_games.json, заметки хранятся отдельно в notes.json.

Код разделён на независимые модули с чёткой ответственностью. Добавление нового функционала не требует переписывания существующего.

Структура проекта:
main.py — главный модуль. Содержит точку входа, основной игровой цикл, диспетчеризацию команд и функцию after_create() для сохранения и тестирования созданных игр.

ui.py — модуль интерфейса. Предоставляет адаптивную систему отрисовки: функции slim_box() и slim_menu() формируют псевдографические рамки, автоматически подстраивающиеся под ширину терминала через shutil.get_terminal_size(). Диапазон ширины: от 30 до 80 символов. Реализован перенос длинных строк по словам через wrap_text().

save_system.py — модуль управления данными игр. Инкапсулирует всю работу с JSON-хранилищем: load_saved_games(), save_game(), delete_saved_game(), edit_game(), show_saved_games(). Также содержит функции валидации ввода: input_int() для чисел и input_bool() для логических значений.

notes.py — модуль заметок. Полноценный CRUD: load_notes(), save_notes(), create_note(), show_notes(), read_note(), edit_note(), delete_note(). ID заметок перенумеруются при удалении. Даты создания и изменения проставляются автоматически через datetime.

genres/ — пакет игровых жанров. Каждый жанр — отдельный класс в своём файле:

hero_villain.py — HeroVillainGenre

friends.py — FriendsGenre

action.py — ActionGenre

survival.py — SurvivalGenre

rpg.py — RPGGenre

horror.py — HorrorGenre

detective.py — DetectiveGenre

earth.py — EarthGenre

medieval.py — MedievalGenre

Каждый класс реализует три метода: create() — сбор параметров от пользователя и возврат game_data, test() — запуск интерактивного тестового сценария, load_and_test(data) — загрузка данных из сохранения и запуск теста.

Метод create() во всех жанрах поддерживает отмену: если пользователь вводит 0 на первом шаге, возвращается None и создание прерывается.

Формат хранения данных
Игры (saved_games.json) — плоский JSON-массив. Обязательные поля: name, platform, genre_type, age, id.

Поля по жанрам:

hero_villain: hero_name, villain_name, hero_skill, villain_skill, location, objective, win_action, lose_action

friends: name_one, name_two, location, history, victory

action: hero_name, hero_weapon, enemy_count, boss_name, boss_ability, location, objective, win_action, lose_action

survival: hero_name, food, water, medkits, danger, days_goal, has_fire, shelter_level

rpg: hero_name, hero_class, starting_gold, quest, companion

horror: hero_name, fear_level, batteries, monster, location, objective

detective: detective_name, location, crime, suspect_count, history, clue_1, clue_2, twist, victory_text, fail_text

Earth: hero_name, weapons, history

Средневековье: hero_name, kor_name, history

Заметки (notes.json) — плоский JSON-массив. Поля: id, title, content, created_at, updated_at.

Поток выполнения:
При запуске вызывается show_start_screen() — приветствие, статистика по сохранённым играм, список последних трёх проектов.

Главный цикл while True обрабатывает шесть команд:

Создание игры: опрос общих данных с возможностью отмены, выбор жанра, вызов genre_classes[choice].create(). Если create() вернул None — отмена и возврат в меню. Иначе вызов after_create() для сохранения и теста.

Загрузка: show_saved_games(), выбор игры, создание объекта жанра, вызов load_and_test().

Редактирование: show_saved_games(), выбор игры, edit_game().

Удаление: show_saved_games(), выбор игры, подтверждение, delete_saved_game().

Заметки: notes_menu() с собственным внутренним циклом.

Выход: завершение программы.

Система тестирования:
Тесты встроены в движок и запускаются через консоль. Каждый жанр имеет уникальный тестовый сценарий с ветвящейся логикой и случайными исходами через random.choice() и random.randint().

Наиболее проработанные тесты:

Выживание — пошаговый цикл с расходом ресурсов, случайными событиями (находки, травмы, воры, опасности), выбором действий на ночь и несколькими вариантами исхода.

Детектив — три этапа расследования с использованием заданных пользователем улик и поворотов. На каждом этапе три варианта действий с разной вероятностью успеха.

Валидация ввода:
Функция input_int(prompt, default) в save_system.py запрашивает ввод пока пользователь не введёт число. Пустой ввод возвращает значение по умолчанию. Используется для всех числовых полей: ресурсы выживания, золото, страх, батарейки, дни.

Функция input_bool(prompt) принимает да/нет/yes/no/1/0 и возвращает True или False. Используется для логических полей: наличие огня, выбор да/нет.

Редактор edit_game() определяет тип текущего значения поля и применяет соответствующую проверку при вводе нового.

Обработка ошибок:
Файловые операции обёрнуты в try-except. Некорректный JSON при загрузке возвращает пустой массив, приложение не падает. Пользовательский ввод проверяется на соответствие ожидаемым значениям. Числовой и булев ввод валидируются с понятными сообщениями об ошибках.

Адаптивный интерфейс:
Ширина терминала определяется динамически через shutil.get_terminal_size(). Диапазон: 30-80 символов. Функция wrap_text() переносит строки по словам, предотвращая обрезание на узких экранах мобильных устройств. Все меню центрируются относительно доступной ширины.

Сборка и зависимости:
Только стандартная библиотека Python: json, os, time, random, shutil, datetime. Внешние зависимости отсутствуют. Приложение запускается на Python 3.x без установки пакетов, включая мобильные окружения Termux и Pydroid.
