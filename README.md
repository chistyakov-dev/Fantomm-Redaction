📘 User Documentation

О проекте

FANTOM REDACTION — текстовый игровой движок с открытым исходным кодом, написанный на Python. Главная цель проекта — показать, что создавать свои мини-игры можно просто и быстро. Программа состоит из понятных скриптов, разобраться в которых сможет даже начинающий разработчик.

Актуальная версия: 3.1.3

Руководство пользователя

Все действия в программе выполняются по выбору пользователя. Программа всегда подсказывает, какую цифру или букву нужно ввести для нужного действия.

Интерфейс

При запуске программы появляется приветственный экран со статистикой созданных игр и списком последних трёх проектов. Интерфейс адаптируется под ширину терминала — работает как на компьютере, так и на смартфоне через Termux или Pydroid.

Главное меню

После загрузки открывается главное меню со всеми возможностями движка.

1 — Создать игру

Запускает мастер создания новой игры. Вы последовательно вводите:

Название игры

Платформу (Мобайл, ПК или Гибрид)

Возрастное ограничение

Уникальный ID (не менее 7 символов)

Жанр и параметры сюжета

Доступно 9 жанров:

Герой и злодей — классическое противостояние

Два друга — история о дружбе и выборе

Экшен — сражения, оружие и боссы

Выживание — управление ресурсами и случайные события

RPG — герой, класс, квесты и спутники

Хоррор — страх, батарейки и монстры

Детектив — расследование с уликами и поворотами

Earth — война с инопланетным разумом

Средневековье — королевство, урожай и дипломатия

После заполнения всех параметров игра готова. Программа предложит сохранить её и сразу протестировать.

2 — Настройки

Раздел с информацией о проекте:

О нас — версия движка, разработчик, ссылка на сайт

Остальные подразделы находятся в разработке и будут добавлены в следующих версиях.

3 — Загрузить игру

Открывает список всех сохранённых игр. Выберите номер — и программа покажет подробную информацию об игре и предложит запустить тестовый режим.

4 — Управление сохранениями

Позволяет просмотреть список сохранённых игр и удалить ненужные. Удаление требует подтверждения — случайно ничего не пропадёт.

5 — Редактировать игру

Открывает сохранённую игру для редактирования. Можно изменить:

Название

Платформу

Возрастное ограничение

ID игры

Все параметры сюжета

Изменения сохраняются в тот же файл.

6 — Заметки

Встроенный блокнот для разработчика. Возможности:

Создание заметок с заголовком и содержимым

Просмотр списка всех заметок

Чтение любой заметки полностью

Редактирование заголовка и текста

Удаление ненужных записей

Заметки хранятся в файле notes.json и остаются навсегда, пока вы сами их не удалите. Удобно записывать идеи для игр, баги, планы по развитию проекта.

0 — Выход

Завершает работу программы.

📙 Developer Documentation

Обзор архитектуры

FANTOM REDACTION построен по модульному принципу. Основной событийный цикл обрабатывает пользовательский ввод и направляет команды в соответствующие модули. Игровые данные сериализуются в JSON-файл saved_games.json, заметки хранятся отдельно в notes.json.

Код разделён на независимые модули с чёткой ответственностью.

Структура проекта:

main.py — главный модуль

Содержит точку входа, основной игровой цикл и диспетчеризацию команд. Здесь же находится рендеринг стартового экрана со статистикой. Импортирует все жанровые классы и модуль заметок.

ui.py — модуль интерфейса

Предоставляет адаптивную систему отрисовки. Функции slim_box() и slim_menu() формируют псевдографические рамки, автоматически подстраивающиеся под ширину терминала через shutil.get_terminal_size(). Диапазон ширины: от 30 до 80 символов. Реализован перенос длинных строк по словам через wrap_text().

save_system.py — модуль управления данными

Инкапсулирует всю работу с JSON-хранилищем игр:

load_saved_games() — загрузка списка

save_game() — сохранение нового проекта

delete_saved_game() — удаление по индексу

edit_game() — редактирование полей

show_saved_games() — отображение списка

notes.py — модуль заметок

Полноценный CRUD для заметок разработчика:

load_notes() — чтение из notes.json

save_notes() — запись в файл

create_note() — создание с автоинкрементным ID

show_notes() — список всех заметок

read_note() — просмотр содержимого

edit_note() — изменение с обновлением updated_at

delete_note() — удаление с перенумерацией

genres/ — пакет игровых жанров

Каждый жанр — отдельный класс в своём файле:

hero_villain.py — класс HeroVillainGenre

friends.py — класс FriendsGenre

action.py — класс ActionGenre

survival.py — класс SurvivalGenre

rpg.py — класс RPGGenre

horror.py — класс HorrorGenre

detective.py — класс DetectiveGenre

earth.py — класс EarthGenre

medieval.py — класс MedievalGenre

Каждый класс реализует три метода:

create() — сбор параметров и возврат game_data

test() — запуск интерактивного теста

load_and_test(data) — загрузка из сохранения и тест

Формат хранения данных
Игры (saved_games.json)

Плоский JSON-массив. Каждый элемент — словарь с обязательными полями:

text
name, platform, genre_type, age, id
Остальные поля зависят от жанра:

Жанр	Поля
hero_villain	hero_name, villain_name, hero_skill, villain_skill, location, objective, win_action, lose_action
friends	name_one, name_two, location, history, victory
action	hero_name, hero_weapon, enemy_count, boss_name, boss_ability, location, objective, win_action, lose_action
survival	hero_name, food, water, medkits, danger, days_goal, has_fire, shelter_level
rpg	hero_name, hero_class, starting_gold, quest, companion
horror	hero_name, fear_level, batteries, monster, location, objective
detective	detective_name, location, crime, suspect_count, history, clue_1, clue_2, twist, victory_text, fail_text
Earth	hero_name, weapons, history
Средневековье	hero_name, kor_name, history
Заметки (notes.json)

Плоский JSON-массив. Каждый элемент:

text
id, title, content, created_at, updated_at
Поток выполнения
show_start_screen() — приветствие, статистика, последние игры

Главный цикл while True — обработка шести основных команд

При выборе «Создать игру» — опрос пользователя, создание объекта жанра, вызов create()

При выборе «Загрузить» — show_saved_games(), выбор игры, load_and_test()

При выборе «Редактировать» — show_saved_games(), выбор игры, edit_game()

При выборе «Заметки» — notes_menu() с собственным внутренним циклом

Система тестирования

Тесты встроены в движок и запускаются через консоль. Каждый жанр имеет уникальный тестовый сценарий с ветвящейся логикой. Используются random.choice() и random.randint() для вариативности.

Наиболее проработанные тесты:

Выживание — пошаговый цикл с расходом ресурсов, случайными событиями, несколькими вариантами исхода

Детектив — три этапа расследования, использует пользовательские улики и повороты

Обработка ошибок

Файловые операции обёрнуты в try-except, некорректный JSON возвращает пустой массив

Пользовательский ввод проверяется, неверные значения выводят понятные сообщения

Числовой ввод валидируется с fallback-значениями по умолчанию

Адаптивный интерфейс:
Ширина терминала определяется динамически через shutil.get_terminal_size(). Диапазон: 30–80 символов. Функция wrap_text() переносит строки по словам, предотвращая обрезание на узких экранах. Все меню центрируются относительно доступной ширины.

Сборка и зависимости:
Только стандартная библиотека Python:

json — сериализация данных

os — работа с файловой системой

time — задержки и метки времени

random — генерация событий

shutil — определение размера терминала

datetime — даты создания и изменения

Внешние зависимости отсутствуют. Приложение запускается на Python 3.x без установки пакетов, включая мобильные окружения Termux и Pydroid.
