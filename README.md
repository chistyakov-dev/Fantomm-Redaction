 FR - это текстовый игровой движок, написанный на ЯП Python с открытым исходным кодом. Главная цель нашего проекта - показать уникальность и простоту в разработке своих мини-игр на базе движка от FR. Наша программа состоит из простых скриптов на Python, поэтому разобраться в ней очень просто.

* В программе все действия выполняются по выбору пользователя, а точнее, что он написал, сама программа говорит, какая цифра или буква, что делает.
  
  Интерфейс: при запуске программы перед вами появляется довольно простой, но красивый интерфейс, с инструкцией для выбора действий.

  Меню: при запуске программы появляется меню движка, где и можно посмотреть основные действия с программой.

  1 - Создать игру: написав цифру 1 в начальном меню вы начнёте создавать свою первую игру. Следуйте инструкциям и тогда у вас всё получится. В конце создания игры вам предложат её сохранить и сразу протестировать.

  2 - Настройки: раздел движка, где есть маленькие подразделы - мощность/о нас/язык/графика. В разделе мощность вы можете настроить требуемый вам режим мощности (бета-режим на 19.06.2026). Раздел "О нас" предлагает пользователю поближе ознакомиться с проектом и его актуальной версией. В разделе "Язык" вы можете изменить язык интерфейса движка. (в разработке на 19.06.2026). Раздел "Графика" предлагает пользователю настроить лучшую графику под его железо. (в разработке на 19.06.2026).

  3 - Загрузить игру: раздел позволяет пользователю загрузить ранее созданную им/ею игру для последующего тестирования.

  4 - Управление сохранениями: отдельный уголок для управления вашими играми. Здесь вы можете посмотреть свои сохранённые программы, а также при необходимости их удалить.

  5 - Редактировать игру: раздел позволяет пользователю изменить сохранённую до этого игру, что помогает вернуться к разработке игры.

FANTOM REDACTION — Dev Docs.

Обзор архитектуры
FANTOM REDACTION представляет собой консольное приложение на Python с монолитной архитектурой. Программа построена вокруг единого событийного цикла, который обрабатывает пользовательский ввод и управляет состоянием через JSON-хранилище. Все игровые данные сериализуются в единый файл saved_games.json, выступающий одновременно базой данных и файлом сохранений.

Структура проекта
Исходный код организован по модульному принципу с разделением ответственности:

Главный модуль содержит точку входа, основной игровой цикл и диспетчеризацию команд. Здесь же находится рендеринг стартового экрана со статистикой.

Модуль интерфейса предоставляет адаптивную систему отрисовки: функции slim_box и slim_menu формируют псевдографические рамки, автоматически подстраивающиеся под ширину терминала через shutil.get_terminal_size(). Реализован перенос длинных строк по словам.

Модуль управления данными инкапсулирует всю работу с JSON-хранилищем: загрузка списка игр, сохранение нового проекта, удаление записей, редактирование полей.

Модули игровых жанров содержат логику создания структуры данных для каждого из девяти поддерживаемых типов игр и соответствующие тестовые сценарии с ветвящимися диалогами.

Формат хранения данных

Все игры сохраняются в плоском JSON-массиве. Каждый элемент массива — словарь с обязательными полями name, platform, genre_type, age, id. Остальные поля определяются типом жанра:

hero_villain: hero_name, villain_name, hero_skill, villain_skill, location, objective, win_action, lose_action

friends: name_one, name_two, location, history, victory

action: hero_name, hero_weapon, enemy_count, boss_name, boss_ability, location, objective, win_action, lose_action

survival: hero_name, food, water, medkits, danger, days_goal, has_fire, shelter_level

rpg: hero_name, hero_class, starting_gold, quest, companion

horror: hero_name, fear_level, batteries, monster, location, objective

detective: detective_name, location, history

Earth: hero_name, weapons, history

Средневековье: hero_name, kor_name, history

Целостность данных обеспечивается на уровне ввода: числовые поля проходят валидацию через isdigit() с fallback-значениями по умолчанию.

Поток выполнения

При запуске приложение вызывает show_start_screen(), которая отображает приветственный экран, статистику по сохранённым играм и список последних трёх проектов. Затем управление передаётся главному циклу while True, обрабатывающему пять основных команд.

Выбор "Создать игру" запускает последовательный опрос пользователя: название, платформа, жанр, возрастное ограничение, ID. После выбора жанра происходит сбор специфичных для сюжета параметров. Сформированный словарь game_data передаётся в save_game().

Выбор "Загрузить" вызывает show_saved_games() для отображения списка, затем загружает выбранную игру и предлагает запустить соответствующий тестовый сценарий. Тестовые сценарии для каждого жанра реализованы как отдельные ветки с интерактивными диалогами.

Выбор "Редактировать" позволяет изменять как общие поля, так и жанрово-специфичные параметры через edit_game().

Система тестирования

Тестовые сценарии встроены непосредственно в движок и запускаются через консольный интерфейс. Каждый жанр имеет уникальный тестовый поток с ветвящейся логикой, использующей случайные события через random.choice() и random.randint(). Наиболее проработанный тестовый модуль — для жанра выживания: реализован пошаговый цикл с расчётом ресурсов, случайными событиями и несколькими вариантами исхода.

Обработка ошибок

Файловые операции обёрнуты в try-except блоки. Некорректный JSON при загрузке приводит к возврату пустого массива, предотвращая падение приложения. Пользовательский ввод проверяется на соответствие ожидаемым значениям с выводом понятных сообщений об ошибках. Числовой ввод валидируется с установкой значений по умолчанию при неконвертируемых данных.

Адаптивный интерфейс

Система рендеринга определяет ширину терминала динамически и ограничивает её диапазоном от 30 до 80 символов. Функция wrap_text обеспечивает перенос строк по словам внутри рамок, предотвращая горизонтальное обрезание контента на узких экранах мобильных устройств. Все меню и информационные блоки центрируются относительно доступной ширины.

Точки расширения

Добавление нового жанра требует реализации в четырёх местах: создание структуры данных в основном цикле, тестовый сценарий, редактор параметров в edit_game(), отображение при загрузке. Иконки жанров задаются через словарь genre_map в функции show_saved_games().

Сборка и зависимости
Проект использует только модули стандартной библиотеки Python: json, os, time, random, shutil. Специфичные внешние зависимости отсутствуют. Приложение может запускаться на любом интерпретаторе Python 3.x без дополнительной установки пакетов, включая мобильные окружения Termux и Pydroid.

Состояние разработки

Версия 3.1.3 находится в активной разработке. Функционал переключения языка интерфейса и настройки графических профилей помечен как бета-режим. Система профилей мощности имеет заглушку в виде информационного экрана без реального влияния на производительность.
